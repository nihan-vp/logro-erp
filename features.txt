ROLE-BASED ACCESS CONTROL
================================================================================

The application supports 4 distinct user roles, each with different access:

  SUPERADMIN  — Full system control (multi-tenant management, company registry)
  ADMIN       — Full company access (all pages)
  ACCOUNTANT  — Financial overview, Finance Hub, Accountant workspace, Projects (read)
  MANAGER     — Projects and Crew management only (no financial data)

================================================================================
SYSTEM-WIDE FEATURES
================================================================================

  AUTH & SECURITY
  - JWT token auth stored in localStorage
  - Role-based page and action visibility
  - Superadmin uses hardcoded credentials (separate from tenant users)
  - Trial / subscription expiry check every 4 seconds
  - Expired subscription → "Trial / Subscription Expired" lockscreen
  - Product key activation form on lockscreen (with sign out option)

  REAL-TIME UPDATES (WebSocket)
  - Socket.IO integration
  - Each company joins its own socket room (tenant isolation)
  - Events:
      · requestsUpdate → triggers auto-refresh in Finance Hub, Accountant, Projects
  - Auto-reconnect on disconnect

  OFFLINE DETECTION
  - OfflineScreen overlay shown when internet connection is lost
  - Restores automatically when connection returns

  TOAST NOTIFICATIONS
  - Success (green), Error (red), Warning (amber) toasts
  - AppToasts component renders them globally

  CONFIRM MODALS
  - Global ConfirmContext for all destructive actions
  - Variants: default, warning, danger
  - Custom title, message, and confirm button label

  NAVIGATION
  - Hash-based routing (#dashboard, #projects, etc.)
  - Browser history pushState / popState support (back button works)
  - Active tab persisted in localStorage (survives page refresh)
  - Deep-linking: navigate directly to a specific project + task from dashboard

  RESPONSIVE DESIGN
  - Desktop: top navigation bar
  - Mobile: fixed bottom navigation bar (icon + label)
  - Card/table layouts adapt to screen size
  - Mobile-specific report card views vs desktop table views

  CURRENCY FORMAT
  - All monetary values formatted as Indian Rupees (₹, en-IN locale)
  - No decimal places for whole rupee amounts

  PAGINATION
  - Finance Hub and Accountant tables: configurable rows per page (5/10/25/50)
  - Setting persisted in localStorage per section
  - Fixed table height to prevent layout shift

DATABASE ARCHITECTURE
================================================================================

  Multi-tenant: each company gets its own isolated MongoDB database.
  Registry DB: stores company metadata (names, status, expiry, keys).
  Tenant DBs:  each named db_<companyname> and holds:
    - users
    - projects
    - tasks
    - expenses
    - attendance
    - payments
    - paymentRequests
    - crew
    - vendors
    - officeFunds
    - officeTransactions
    - documents
